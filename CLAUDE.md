## Project Overview

Rhino Mocks is a dynamic mock object framework for .NET. The main NuGet package published is `Rhino.Mocks.Light` (version 4.0 alpha). It targets `net461`, `net48`, and `net8.0`. The core dependency is **Castle.Core 4.3.1** (DynamicProxy) for runtime proxy generation.

## Commands

### Build
```sh
dotnet build --configuration Release
```

### Test (all tests)
```sh
dotnet test --no-build --configuration Release
```

### Test (single test project)
```sh
dotnet test Rhino.Mocks.Tests/Rhino.Mocks.Tests.csproj --configuration Release
```

### Test (single test by name filter)
```sh
dotnet test --configuration Release --filter "FullyQualifiedName~TestClassName"
```

### Pack NuGet
```sh
dotnet pack Rhino.Mocks/Rhino.Mocks.csproj -c Release -o artifacts /p:Version=4.0.0
```

> **Note:** CI runs on `windows-latest`. Some tests (in `Rhino.Mocks.Windows.Tests`) and features may be Windows-specific. Remoting-based mock files are excluded from the `net8.0` build.

## Architecture

### Core State Machine

The central abstraction is a **Record → Replay → Verify** state machine per mock object:

- **`MockRepository`** (`Rhino.Mocks/MockRepository.cs`) — the main entry point. Creates mock objects and manages state transitions (`Replay()`, `Verify()`, `BackToRecord()`).
- **`IMockState`** (`Interfaces/IMockState.cs`) — interface implemented by all state classes. Each mock object holds a current state.
- State implementations in `Impl/`:
  - `RecordMockState` / `RecordDynamicMockState` / `RecordPartialMockState` — recording phase, captures expectations
  - `ReplayMockState` / `ReplayDynamicMockState` / `ReplayPartialMockState` — replay phase, matches actual calls to expectations
  - `StubRecordMockState` / `StubReplayMockState` — stub variants (no strict expectation verification)
  - `VerifiedMockState` — post-verify terminal state

### Proxy & Interception

- Mock objects are runtime proxies generated by **Castle.DynamicProxy**.
- **`RhinoInterceptor`** (`Impl/RhinoInterceptor.cs`) intercepts every method call and delegates to the current `IMockState`.
- **`ProxyInstance`** (`Impl/ProxyInstance.cs`) wraps the Castle proxy and holds associated state.
- For `net461`/`net48`, remoting-based mocking is also supported via `Impl/RemotingMock/` (excluded on `net8.0`).

### Expectations & Constraints

- **`Expectations/`** — defines what calls are expected and how many times:
  - `AbstractExpectation` — base with return value, repeat counts, callbacks
  - `ArgsEqualExpectation`, `AnyArgsExpectation`, `ConstraintsExpectation`, `CallbackExpectation`
- **`Constraints/`** — argument matchers used in `ConstraintsExpectation`:
  - `Is`, `Text`, `List`, `Property`, `PublicField`, `IsArg`, `ListArg`, `TextArg`
  - All inherit from `AbstractConstraint`

### Method Recorders

- **`MethodRecorders/`** — store and match recorded expectations against actual calls:
  - `UnorderedMethodRecorder` — default, calls can occur in any order
  - `OrderedMethodRecorder` — enforces strict ordering (used inside `using (mocks.Ordered()) { }`)
  - `RecorderChanger` — handles switching between ordered/unordered scopes

### Two Public APIs

1. **Classic Record/Replay API** — use `MockRepository` directly: create mock → set expectations → `ReplayAll()` → exercise → `VerifyAll()`
2. **AAA (Arrange-Act-Assert) API** — extension methods in `RhinoMocksExtensions.cs` (compiled under `#if DOTNET35`): `mock.Expect(x => x.Method())`, `mock.Stub(...)`, `mock.AssertWasCalled(...)`

### Assembly Signing

The assembly is strong-named using `ayende-open-source.snk`. The tests project is granted `[InternalsVisibleTo]` via a `PublicKey` attribute set during the build.

## CI / Release

- **CI** (`.github/workflows/main.yml`): triggers on push/PR to `master`, runs `dotnet build` + `dotnet test` on `windows-latest`.
- **NuGet publish** (`.github/workflows/publish-nuget.yml`): triggers on tags matching `v*`, packs and pushes to NuGet.org using Trusted Publishing. Tag must be `v<version>` (e.g. `v4.0.1`).
